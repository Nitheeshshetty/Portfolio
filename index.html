The HANA to GCP Migration project transitioned enterprise data from SAP HANA to Google Cloud Platform (GCP) for better scalability, performance, and cost efficiency. It involved designing GCP BigQuery tables, optimizing stored procedures, and ensuring seamless data transformation. Performance tuning and query optimization enhanced processing efficiency. The migration enabled advanced analytics and reporting, improving business decision-making.

Network Inventory Transformation

The Network Inventory Transformation project migrates XNG to ARM (Amdocs Resource Management system) to enhance network asset management. XNG lacks scalability, automation, and accurate resource tracking, making migration essential. ARM provides a centralized, efficient, and future-ready system for better inventory control. The migration process leverages Kafka streaming to ingest data from the application directly into GCP for real-time ETL processing. An Airflow DAG was developed to orchestrate and manage the data pipeline efficiently. Additionally, GCP BigQuery stored procedures were implemented to cleanse, validate, and enhance the data, ensuring accuracy and consistency.

GCP BigQuery Oracle SQL Python Unix Airflow
